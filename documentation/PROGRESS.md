## 核心设计原则

### 1. 配置驱动开发
所有游戏参数集中在 `GameConfig.js` 和 `MenuConfig.js` 中：
- **GameConfig**: 敌人速度、射击冷却、波次参数、UI 位置等
- **MenuConfig**: 菜单样式、颜色、按钮大小、文字位置等
- **优势**: 修改游戏参数无需触碰逻辑代码，易于调试和微调

### 2. Manager 模式（职责分离）
每个管理器只负责一个域：
- **AudioManager**: 音乐和音效的播放/暂停/停止
- **ScoreManager**: 分数计算和高分持久化
- **EffectsManager**: 所有视觉效果（闪烁、文字、粒子）
- **InputManager**: 统一处理键盘/虚拟按钮/触摸输入
- **BulletManager**: 子弹的生成、销毁、碰撞
- **UIManager**: HUD 元素和虚拟按钮，委托菜单给 MenuManager
- **MenuManager**: 菜单的显示/隐藏/栈管理

### 3. 菜单系统分层
```
Depth 100: 菜单元素（按钮、文字）       - 最顶层
Depth 90:  背景遮罩（70% 不透明）      - 阻止穿透
Depth 0:   游戏对象（玩家、敌人、HUD）  - 底层
```
这确保菜单永不被游戏对象覆盖，同时菜单栈支持未来的嵌套菜单。

## 会话 7 - 敌人 AI 方案1：预测性射击系统

**时间**: 2025-11-24
**影响**: High
**成果**: 智能敌人AI + 模块化架构 + 测试工具

### 关键成就
- **方案1：预测性射击系统** - 完整实现
  - Wave 1保留随机射击（新手引导）
  - Wave 2-5启用瞄准射击，渐进难度（30%→90%）
  - 预判系统（考虑玩家速度，0.3系数）
  - 瞄准精度80%（保留20%偏差）
- **模块化射击架构** - 为方案2/3预留扩展
- **视觉警告系统** - 红色边框+敌人放大（200ms）
- **GM测试按钮** - 一键击杀敌人，快速测试

### 架构改进
- **射击模式路由器**: `GameConfig.ENEMY.SHOOTING.CURRENT_MODE`
  - 支持切换：RANDOM / AIMED / FORMATION / POSITIONING
  - 每个模式独立实现，零耦合
- **波次同步机制**: GameScene ↔ BulletManager
  - `setCurrentWave()` 确保AI感知当前难度
- **配置驱动AI**: 所有参数可调（瞄准概率、预判系数、精度）

### 技术决策
- **为什么不直接改原有方法？**
  - 保留原始逻辑（RANDOM模式）便于回退测试
  - 模块化设计利于未来扩展（方案2编队射击、方案3智能站位）
- **视觉警告为什么用边框而不是tint？**
  - 敌人是绿色纹理（0x00ff00），红色tint会变黑色（颜色相乘）
  - 独立graphics边框不受纹理颜色影响，跟随敌人移动
- **为什么波次从2开始启用瞄准？**
  - Wave 1作为新手引导期，保持简单直线射击
  - 渐进难度曲线避免突然难度跳跃

### Bug 修复
- **游戏结束射击未停止**: `endGame()` 中添加 `enemyFireTimer.paused = true`

### 代码规模
- GameScene: 393行 → 438行 (+45行) - 波次同步+GM按钮
- BulletManager: 120行 → 305行 (+185行) - 核心AI逻辑
- UIManager: 420行 → 446行 (+26行) - GM按钮
- GameConfig: 102行 → 135行 (+33行) - 模块化配置
- **总计**: +289行净增

### 关键洞察
- 配置模块化不仅利于参数调整，更是多方案架构的基础
- 视觉反馈的物理实现方式需考虑现有资源特性（纹理颜色、graphics特性）
- 测试工具（GM按钮）显著提升开发效率，快速验证各波次AI行为

### 性能影响
- **内存**: 无明显增加（~24.5 MB）
- **FPS**: 保持60+稳定
- **计算开销**: 可忽略（瞄准计算仅在射击时触发，~1次/秒）
